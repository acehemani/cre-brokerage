## v2 Enhancements

### 1. DEAL PIPELINE — Make it Interactive
- **Drag-and-drop Kanban**: Cards should be draggable between stages. When a card is dropped into a new stage, update the deal's stage in local state and show a toast: "Moved [deal name] to [stage]".
- Use a lightweight drag library (dnd-kit or @hello-pangea/dnd) or native HTML5 drag. Keep it smooth.
- **Filter bar** above the kanban: filter by property type (Office, Retail, Industrial, Multifamily, Mixed-Use) and broker name. Show active filter pills.
- **Search**: text search across deal names and addresses.

### 2. DEAL DETAIL PAGE (New Screen)
- Instead of the slide-out panel, clicking a deal navigates to `/deals/:id` — a full detail page.
- **Hero section**: property photo (use placeholder from picsum.photos or a gradient), property name, type badge, address, key metrics row (price, SF, cap rate, NOI).
- **Tabs**: Overview | Documents | Financials | Activity
  - **Overview**: deal summary, key dates, broker info, map placeholder
  - **Documents**: same doc list from v1 slide-out but with the drag-drop upload zone from Document Intelligence (scoped to this deal)
  - **Financials**: mock P&L table (revenue, expenses, NOI, debt service, cash flow) + a simple Recharts bar chart of trailing 12-month NOI
  - **Activity**: timeline from v1 but richer — group by date, show agent vs human icons, add "Add Note" button (saves to local state)
- **Back button** returns to pipeline with scroll position preserved.

### 3. COMMAND CENTER — Add Interactivity
- "Needs Your Attention" cards: clicking the action button should navigate to the relevant deal detail page.
- Add a **"Quick Actions"** row: buttons for "Add New Deal", "Upload Document", "Generate Report" — each opens a modal with a mock form that shows a success toast on submit.
- Activity feed items that mention a deal should be clickable → navigate to that deal.

### 4. DOCUMENT INTELLIGENCE — Enhance the Demo
- After the drag-drop simulation completes, add a **"Save to Deal"** button that opens a dropdown of deals. Selecting one shows a toast: "Document saved to [deal name]".
- Add **document type filter** tabs: All | OM | Rent Roll | Financial | Inspection
- Add a **processing stats bar** at the top: "142 documents processed this month | 98.6% accuracy | Avg processing time: 4.2 seconds"

### 5. ANALYTICS — Deepen the Charts
- Add a **date range selector** (preset buttons: 30d, 90d, 6mo, 1yr) that visually filters chart data.
- New chart: **Broker Leaderboard** — horizontal bar chart showing deals per broker.
- New chart: **Deal Type Distribution** — donut/pie chart by property type.
- Make the ROI callout **animated** — numbers count up from 0 when the page loads.

### 6. MORNING BRIEFING — Polish
- Add a **"Regenerate"** button that shows a brief loading spinner, then swaps in slightly different mock briefing content (just rotate between 2 sets of data).
- Action items should be checkable — clicking marks them done with a strikethrough animation.
- Add a **"Share as PDF"** button (mock — just shows toast).

### 7. GLOBAL IMPROVEMENTS
- **Toast system**: Create a shared toast component (bottom-right, auto-dismiss after 3s, supports success/error/info variants). Replace all inline toast implementations.
- **Loading skeletons**: Add skeleton placeholders on each page that flash briefly (200ms) before content appears, to simulate data loading.
- **Breadcrumbs**: Add breadcrumb nav at the top of the content area (Home > Deal Pipeline > Westheimer Office Tower).
- **Keyboard shortcuts**: `1-5` keys navigate to screens, `Esc` closes modals, `/` focuses search on pipeline.
- **Page titles**: Update `document.title` on each route change.
